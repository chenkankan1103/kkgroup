# 🌐 動態 IP 配置指南 - 完全免費解決方案

## 問題

GCP 上的外部 IP 會定期變更（浮動 IP），導致 Google Sheet 中硬編碼的 API 地址失效。每次 IP 變更後，需要：
1. 查詢新 IP
2. 修改 Apps Script 代碼
3. 重新部署

## ✅ 完全免費的解決方案

直接在 **Google Sheet** 上建立「系統配置」表，Apps Script 會自動讀取 API IP。當 IP 變更時，**只需更新 Sheet 中的一個單元格**！

### 優點
- ✅ **完全免費** - 無需購買靜態 IP
- ✅ **無代碼修改** - 不需要更新 Apps Script
- ✅ **即時生效** - 修改配置立即應用
- ✅ **回退保護** - 有預設值防止意外

---

## 設置步驟

### 第 1 步：在 Google Sheet 中新增工作頁

1. 打開你的 Google Sheet
2. 在底部點擊 **「+」** 新增工作頁
3. 將工作頁名稱改為：**`系統配置`**（必須完全相同）

```
┌─────────────────────────┐
│ 玩家資料 │ 系統配置 ✓ │ + │
└─────────────────────────┘
```

### 第 2 步：設置配置表的表頭

在「系統配置」工作頁的第 1 行，填入：

| A1 | B1 |
|----|-----|
| 設定名稱 | 設定值 |

```
推薦：
A1 輸入：Setting Name
B1 輸入：Value
（或用中文）
```

### 第 3 步：添加 API 端點配置

在第 2 行填入：

| A2 | B2 |
|----|----|
| API_ENDPOINT | http://35.206.126.157:5000 |

![設置表範例]

```
A2: API_ENDPOINT
B2: http://35.206.126.157:5000
```

### 第 4 步：測試連接

1. 打開「🔄 同步工具」菜單
2. 點選「✅ 檢查 API 連接」
3. 如果看到 ✅ 連接正常，配置成功！

## 使用流程

### 當 GCP IP 變更時

1. 查詢新 IP（例如：`35.209.xxx.xxx`）
2. 打開「系統配置」工作頁
3. 在 B2 單元格更新新 IP：`http://新IP:5000`
4. 完成！無需修改代碼

```
舊配置：
B2 = http://35.206.126.157:5000

IP 變更後：
B2 = http://35.209.123.456:5000
```

5. 可選：點選「✅ 檢查 API 連接」驗證連接

## 故障排除

### ❌ 找不到「系統配置」工作頁

**症狀**：檢查連接時顯示「找不到系統配置工作頁」

**解決**：
- 確認工作頁名稱是否完全正確：**`系統配置`**
- 不能有多餘空格或特殊字符
- 工作頁必須在同一個 Google Sheet 內

### ❌ 顯示預設 IP（舊 IP）

**症狀**：API 連接失敗，日誌顯示使用預設值

**解決**：
- 檢查「系統配置」表是否有至少 2 行數據
- 確認 A2 = `API_ENDPOINT`（大小寫區分）
- 確認 B2 填入了完整的 URL（包括 `http://` 或 `https://`）
- URL 必須是：`http://IP:5000`

### ❌ API 連接失敗

**症狀**：點「檢查 API 連接」時顯示紅色 ❌

**解決**：
- 確認新 IP 正確無誤
- 檢查 Flask 伺服器是否運行：`sudo supervisorctl status bot-and-api`
- 檢查防火牆：在 GCP 中確認 5000 埠已開放

## 工作原理

修改後的 Apps Script 會：

1. 在每次執行時調用 `getAPIEndpoint()` 函數
2. 自動從「系統配置」表讀取 `API_ENDPOINT`
3. 如果找不到配置，使用預設值（舊 IP）
4. 所有 API 呼叫都使用動態讀取的 IP

```javascript
// 修改前（硬編碼）
const API_ENDPOINT = "http://35.206.126.157:5000";

// 修改後（動態）
const apiEndpoint = getAPIEndpoint();  // 從 Sheet 讀取
UrlFetchApp.fetch(`${apiEndpoint}/api/sync`, options);
```
