## 🖼️ KK園區圖片功能使用指南

### 📋 功能概覽

機器人現在支持三種圖片相關功能：

#### 1. **🎨 生圖** (文字生圖)
- **關鍵詞**: 「生」、「生圖」、「生成圖」、「畫」
- **用法**: `@機器人 生圖 [圖片描述]`
- **例子**:
  ```
  @機器人 生圖 KK園區的監控室，五台螢幕閃著紅色警告，桌上有吃剩的泡麵
  @機器人 畫一張 1990 年代風格的電腦介面
  ```
- **狀態**: ✅ **完全支持** (使用 gemini-2.5-flash-image 模型)

#### 2. **🎨 編圖** (圖片編輯)
- **關鍵詞**: 「編」、「P圖」、「編輯圖」、「改」、「修改」、「調整」、「改圖」
- **用法**: 上傳圖片 + `@機器人 編圖 [編輯指令]`
- **例子**:
  ```
  [上傳排行榜截圖]
  @機器人 編圖 把背景調暗，加上更多的螢光綠效果
  
  [上傳Team Logo]
  @機器人 改 把左上角的標誌換成骷髏頭
  ```
- **狀態**: ✅ **完全支持** (使用 gemini-2.5-flash-image 模型)

#### 3. **🔍 看圖** (圖片分析)
- **關鍵詞**: 「看」、「看圖」、「分析圖」、「檢查圖」、「掃描圖」
- **用法**: 上傳圖片 + `@機器人 看圖 [分析問題]` 或 `@機器人 看圖`
- **例子**:
  ```
  [上傳轉帳截圖]
  @機器人 看圖 這張圖片的金額和時間是多少？看起來真的嗎？
  
  [上傳購物證明]
  @機器人 分析圖 確認這是真的購物單據還是P圖
  ```
- **狀態**: ✅ **完全支持** (使用 gemini-2.5-flash 模型)

---

### 🛠️ 技術細節

#### 支持的圖片格式
- PNG, JPG/JPEG, GIF, WebP

#### 模型選擇
- **分析 (看圖)**: `gemini-2.5-flash` (快速視覺理解)
- **生成/編輯 (生圖/編圖)**: `gemini-2.5-flash-image` (專業圖片生成模型)

#### 歷史記錄
- 所有圖片操作記錄存放在 `data/image_history/user_[ID].json`
- 每個使用者最多保留100筆記錄

### ❓ 常見問題

**Q: 所有功能都可以使用了嗎？**
A: 是的！生圖、編圖、看圖三個功能已全部啟用並使用正確的 Gemini API 模型。

**Q: 圖片分析會洩露隱私嗎？**
A: 圖片傳送到Google API進行分析。敏感內容（如個人ID、銀行卡號）不建議直接傳送。

**Q: 如何查看操作歷史？**
A: 歷史記錄自動保存在GCP伺服器，可通過 `data/image_history/` 目錄查看。

**Q: 生成的圖片有浮水印嗎？**
A: 是的，所有 Gemini 2.5 Flash Image 生成的圖片都包含隱藏的 SynthID 浮水印，用於版權保護。
