## 🔄 數據流對比

### v1.0 架構流

```
       Google Sheets (手動編輯)
            ↓
       Apps Script
            ↓
       Flask API v1.0
            ↓
    sheet_sync_manager
         (固定欄位列表)
            ↓
   SQLite (手動修改 schema)
```

**問題**：
- 欄位需要在 Python 中硬編碼
- 新增欄位需要重啟
- 容易出現不匹配

### v2.0 架構流

```
       Google Sheets (真理來源) ← 表頭就是欄位定義！
            ↓
       Apps Script
            ↓
    Flask API v2.0 (自動檢測表頭)
            ↓
    sheet_sync_manager_v2
    ┌─ 檢查 schema
    ├─ 發現新欄位
    ├─ 推斷欄位類型
    └─ 執行 ALTER TABLE
            ↓
   SQLite (自動同步 schema)
```

**優勢**：
- 欄位完全來自 SHEET 表頭
- 無需重啟，自動 ALTER TABLE
- Schema 永遠與 SHEET 同步

## 💡 實用提示

### Tip 1：欄位命名規範

```
✅ 推薦（便於自動推斷）：
- user_id, player_id
- level, xp, exp
- kkcoin, gold, coin
- hp, stamina, health
- created_at, updated_at
- is_active, has_badge

❌ 避免（難以推斷）：
- 玩家等級（改為 level）
- 金幣數（改為 kkcoin）
- 是否激活（改為 is_active）
```

### Tip 2：欄位分組

```
在 SHEET Row 1 使用分組標題：
[玩家基本, 玩家基本, 遊戲數據, 遊戲數據, ...]

便於視覺組織，同時不影響 Row 2 的表頭
```

```
直接用 TEXT，以後可以改
v2.0 支持修改類型（改名或改推斷規則）
完全安全
```
